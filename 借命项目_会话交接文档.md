## 0. 项目一句话说明

把小说《借命》第一集的 30 个分镜,用 ComfyUI + Wan 2.2 做成"首尾帧 → 短视频"批量生成。GUI 是 `comfy_wan22_gui.py`,单文件 4842 行 tkinter。

## 2. 程序的关键类和数据流

```
用户
 │
 ▼
[ComfyBatchGUI]  ←── 主 GUI 类,含 self.scenes 列表和所有按钮回调
 │
 ├─ self.scenes = [{name, start, end, prompt, gpt_prompt, color, status, video_path}, ...]
 │
 ├─ AssetLibrary(self.asset_lib)
 │    ├─ rescan() 扫素材库目录(递归),文件名去扩展名即关键字
 │    ├─ match(text) 对任意文本返回 [(关键字, 路径)]
 │    └─ 读同目录的 aliases.json 支持别名映射
 │
 ├─ ColorAnchors(self.color_anchors)
 │    └─ inject(prompt, color_id) 在 prompt 前缀加色板文字
 │
 └─ GPTWebController(self.gpt_ctrl)  ←── 用 selenium 驱动 Chrome 操作 GPT 网页
      ├─ start() 启动 Chrome,三种 mode: attach / standalone / temp
      ├─ send(text) 填入 #prompt-textarea 并发送
      ├─ upload_files(paths) send_keys 塞进 input[type=file]
      ├─ wait_for_images() 轮询等图
      ├─ download_last_images() 下载
      ├─ new_chat()  ★ v0.10 新增
      └─ count_composer_attachments()  ★ v0.10 新增
```

**批量生图主函数:`_batch_gpt_images`(当前在 4662 行附近)**
v0.8 起拆成"两次调用":第 1 次只要首帧,第 2 次基于同对话上下文要尾帧。
v0.10 起在每个场景开头 `new_chat()`,上传参考图后做数量校验。

## 3. 当前已知问题与决策记录

### Q1: 匹配率只有 13/30,是 bug 吗?
**A: 不是 bug,是数据问题。** (2026-04-23 定论)

分解 13/30:
- 7 个真·空镜(01/02/11/14/15/24/28)正确没匹配 ⬜
- 13 个有角色的场景正确匹配 ✅
- 6 个"中年男子"戏份(03/05/09/10/12/13)没匹配,因为素材库里没这个角色的图
- 4 个手部特写(04/19/25/27)没匹配,因为 prompt 只写"一只手"没写人名

**修法**:
- 做法 A:素材库加一张 `中年男子.png`(小说里他叫什么名就用什么名),同目录加 `aliases.json`(见第 5 节)
- 做法 B:改 CSV,把"年轻男人的手"替换成"林默的手"等具体名字

### Q2: 批量生图时人物串味 / 画风漂移
**A: v0.10 已解决。** 每个场景开头 new_chat(),彻底清上下文。
- 副作用:每个场景都失去了"让 GPT 看上一张再画下一张"的上下文延续性,但因为拆成两次调用的流程里首帧和尾帧仍在同一对话,所以尾帧的一致性不受影响。

### Q3: 上传参考图有时会有上一场景的残留
**A: v0.10 已解决。** `_upload_with_verify` 上传后检查 `count_composer_attachments()`,数量不对就 new_chat 重试一次。

## 6. 给下一个会话的 Claude:怎么迅速接手?

如果你(未来的 Claude)看到这份文档,用户多半又要改这个 GUI。操作流程:

1. **先读这份文档 + README_GUI.md** —— 获取项目上下文
2. **看版本号**(`comfy_wan22_gui.py` 第 96 行附近的 `__version__`),对照本文第 4 节判断当前是 v 几
3. **git clone 仓库到本地再改** —— 文件 4842 行,直接发到上下文会占满
4. **改动点集中在几处**:
   - `class GPTWebController`(~775 行)——Chrome + selenium 层
   - `class ComfyBatchGUI`(~1535 行)——UI + 主逻辑
   - `_batch_gpt_images`(~4662 行)——批量生图主循环
   - `_run_single_gpt_scene`(~3083 行)——单场景生图
   - `_auto_match_assets_to_scenes`(~3011 行)——素材匹配统计
5. **CSV 不要轻易改结构** —— 列顺序是 scene_name / start_image / end_image / prompt / gpt_prompt / color,GUI 读写都按这个顺序
6. **不要在 CSV 里直接写 aliases** —— aliases 机制是 AssetLibrary 扫 JSON,和 CSV 解耦
