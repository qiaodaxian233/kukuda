## 🎯 一句话项目概述

我在做一个叫**《借命》**的 AI 首尾帧动漫剧(Wan 2.2 + GPT 生图)。输入是小说章节,输出是**每集 30 格分镜**,用自己写的 Python GUI 工具(v0.8,横屏 16:9 默认)批量跑 GPT 画图 + Wan 2.2 动态视频。**需要 Claude 干的是解说稿 + 分镜表 + CSV 三件套**。

## 🔴 硬性规则(违反会导致整集废掉)

### 规则 0:**横屏 16:9 是默认**(v0.8 起)
- **所有 `gpt_prompt` 末尾必须写"横屏 16:9"**(不再是 "竖屏 9:16")
- 画面描述里的构图词改为**横版**:广角横幅 / 平视横构图 / 侧逆光横景 等
- 首尾帧微变化也要适合横屏(左右横摇/远近推移/环境光变,**不适合**上下摇镜)
- CSV 导入工具后,默认 `width=960 height=544`
- 如果章节需求明确要求竖屏(比如社媒短视频),用户会显式说明,那时再改回 9:16

**工具 v0.8 新流程**:批量 GPT 生图是**两次请求**(先发首帧 → 拿到 → 再发尾帧,基于首帧生成)。
这对 Claude 的影响是:**gpt_prompt 必须把首帧和尾帧的差异说清楚**,因为工具发第二次请求时会告诉 GPT "基于刚才那张首帧生成尾帧",所以你写的"首帧:xxx;尾帧:同一构图,xxx" 这个微变化描述会被工具拿来精确指导尾帧。**写得越清楚,尾帧越对**。

### 规则 1:血腥内容全部弱化
**这本小说太监了 5-6 年,原因就是早期过血腥。** 所以:
- ❌ 不拍:血洞特写、断肢、剁尸、喂狗、几刀几刀、尸体分尸、爆头、肠子/脑浆、强奸、虐童
- ✅ 替代:
  - 致命瞬间 → 仰倒+瞳孔涣散+雨落面颊
  - 砍人 → 刀光剪影+金属寒光+速度线
  - 枪击 → 血花用光效,不用红色液体特写
  - 尸体 → 不进镜,直接跳转场
- **原则**:有杀气、有死亡的暗示即可,但不要具体视觉细节

### 规则 2:CSV 必须 6 列,列名固定
```
scene_name, start_image, end_image, prompt, gpt_prompt, color
```
- `scene_name`:带序号,如 `01_雨夜远景`
- `start_image` / `end_image`:留空,批量 GPT 生图会回填
- `prompt`:给 Wan 2.2 的动作指令(**短**,20 字内,动词优先)
- `gpt_prompt`:给 GPT 画图的详细描述(100-180 字,含构图/主体/环境/风格/横屏)
- `color`:色板 ID(见下表)

**必须用 `csv.writer(f, quoting=csv.QUOTE_ALL)` + `utf-8-sig` 编码**,不然工具导入会崩。写完用 `csv.DictReader` 反向读一次校验。

### 规则 3:角色一致性不能跨集漂移
见下方「角色库」。每集生成后,**角色描述必须逐字复制**,不能自己改。

### 规则 4:gpt_prompt 里的人物名**必须与素材库文件名对齐**
工具跑 GPT 生图时,会用 `asset_lib.match()` 扫描 gpt_prompt 文本,**把匹配到的人物参考图自动上传给 GPT**。所以:
- ✅ 写 "眼镜反派(无框眼镜,白衬衫黑风衣)" → 工具匹配到 `眼镜反派.png`,上传给 GPT 作参考
- ❌ 写 "狐哥(无框眼镜...)" → 素材库没有 "狐哥",**什么都不上传**,GPT 随机画一个不一样的人

**Claude 默认用「角色库」表里的精确文件名**。如果章节出现新角色,先向用户报告新角色设计,等用户建好素材文件,再生成 CSV。

### 规则 5:**纯环境空镜必须显式声明「无人物」**,否则 GPT 会强加人物进去
工具给每条 prompt 自动追加"保持人物一致"类似的指令。对**纯空镜格子**(城市远景、天空、空巷、建筑立面等),这条指令会反向诱导 GPT 添加剪影/背影人物,破坏空镜意图。

**判断标准**:gpt_prompt 里**没有**任何人物关键词(林默/林渊/苏郁/眼镜反派/刀疤青年/妈妈/男子/女子/青年/中年/手/脸/背影/侧影/身影/...)

**对这类格子,gpt_prompt 末尾必须追加一句:**
```
【⚠️ 空镜硬性要求】画面中绝对不得出现任何人物、剪影、背影、手或人影,只拍环境/建筑/天气/物品。
```

工具代码里也有 has_people 分支检测自动切换指令模板,两道保险。

**第一集的 6 个空镜格参考**:01 雨夜远景 / 02 标题浮现 / 14 夜转晨 / 15 晨光初现 / 24 白昼巷尾 / 28 中天大厦

### 规则 5:gpt_prompt 必须写明"**同一场景首尾两帧**"能做的微小变化
工具生成的是 **Wan 2.2 首尾帧动画** — 首帧和尾帧必须是**同一机位、同一构图、同一场景**的两个连续瞬间,只有主体的**细微动作/表情/光线变化**。否则 GPT 会画出两张完全不同场景的图,Wan 无法做平滑过渡。

**写 gpt_prompt 时,末尾必须指明首帧 → 尾帧的微小变化**:
- ✅ 好例子(动作过渡):"首帧:男子站立俯视街景,微低头;尾帧:同一构图,男子缓慢抬头,目光锐利"
- ✅ 好例子(镜头推移):"首帧:镜头距建筑较远,雨丝细;尾帧:同一机位,镜头略推近,雨势更大"
- ✅ 好例子(光影变化):"首帧:霓虹灯亮;尾帧:同一画面,一块霓虹熄灭,另一块亮起"
- ❌ 坏例子:只写一段场景描述,不给首尾帧差异 → GPT 会自由发挥两张不同的图
- ❌ 坏例子:首尾变化过大(场景切换、换机位、换构图)→ Wan 2.2 做不出平滑动画

**纯环境空镜(无人物格)的写法**:
- 必须明确写 **"纯环境空镜,画面中无任何人物,无招牌文字,中国都市"**
- 首尾帧变化只能用:镜头微推移、雨势变化、云层流动、光线闪烁等环境变化

**规避地域刻板印象**:
- 写赛博朋克夜景时,**必须明确"中国都市"**,不然 GPT 会默认生成日本东京(霓虹+日文招牌)
- 不写具体招牌文字内容(让 GPT 只画光晕不画字)

## 📐 分镜规范

### 每集默认 30 格
- 可根据章节长度调整到 25-40 格之间
- 章节短时可扩(加回忆闪回、表情反应)
- 章节长时可压(跳过次要情节)

### 色板分配参考(第一集的节奏,可复用)
- 冷开场(动作/悬疑):13 格左右
- 过渡:2 格
- 情感高潮:7 格左右
- 推进+反差:5-7 格
- 黄昏钩子收尾:2 格

### `prompt` 字段(Wan 2.2 动作指令)
- **20 字内,动词优先**
- 镜头 + 主体动作 + 环境 三要素
- ✅ 好例子:`镜头缓慢推近 雨丝斜飞 角色抬头`
- ❌ 坏例子:`情绪变化` `氛围浓郁` `画面感十足`(抽象词)

### `gpt_prompt` 字段(GPT 画图指令)
- **120-220 字**,中文逗号
- 必含(按顺序):
  1. **镜头角度** + **机位**(俯视/仰视/平视/侧逆光...)
  2. **主体描述**(含角色一致性元素,或"纯环境空镜无人物")
  3. **环境**(场景+地域,明确"中国都市"避免日本化)
  4. **光线/色温**
  5. **风格**(电影感/写实/极简/纪录片感等)
  6. **"横屏 16:9"**(v0.8 起默认)
  7. **首尾帧微变化**:"首帧:xxx;尾帧:同一构图,xxx"(必须!见规则 5)
- 禁用词:`electric` `dynamic` `epic` 这类 AI 味重的英文形容词
- 禁用:具体招牌文字内容(比如"写着'小龙餐厅'的招牌"→ 改为"霓虹招牌光晕,字迹模糊")

## 📝 解说稿规范

### 风格:电视剧解说 / 抖音短剧旁白
- 第三人称全知视角
- 多用语气词:`"没想到"` `"然而"` `"结果呢"` `"此时"` `"谁能想到"`
- 关键节点加**粗体**+数字(适合画面弹字)
- 可加小标题分隔线(方便口播换气)
- 结尾留钩子

### 字数 / 时长
- 每集 **1500-2500 字**(约 3-5 分钟口播)
- 每格分镜对应 **6-10 秒**口播(≈ 30-50 字)

### 节奏
- 冷开场:快、简短、制造悬念
- 情感戏:慢、留白、让观众代入
- 反转:突然加速,`"——"` 或 `"此时"` 做节拍

### 禁用
- ❌ 不要"家人们""老铁"这种直播感用语(除非我明确要)
- ❌ 不要用"震惊体"标题
- ❌ 不要剧透后面章节

### 血腥处理(与分镜规则一致)
- 原文砍 50 刀 → "浑身是伤"
- 原文剁碎装袋 → 整段删除
- 原文爆头 → "再也支撑不住,倒了下去"
- 原文强奸/虐待 → 跳过或用"遭遇不测"一笔带过

## 🛠 工具对接(v0.8 信息,不用改)

- CSV 导入后,工具会按 `color` 字段自动注入色板前缀到 `gpt_prompt`
- **v0.8 关键**:工具发 GPT 时是**两次请求**:
  - 第一次:`[你写的 gpt_prompt] + 系统追加"只要一张首帧图(横屏 16:9)"` → 拿首帧
  - 第二次:`[同样的 gpt_prompt] + 系统追加"基于刚才的首帧,生成尾帧(同一场景同一构图)"` → 拿尾帧
  - 所以你写的"首帧:xxx;尾帧:同一构图,xxx" 这段描述在**两次请求里都会出现**,首帧请求靠它定调,尾帧请求靠它做差异
- 工具发送 Wan 时:色板前缀 + prompt → 生成视频
- 默认分辨率:960×544(横屏 16:9);方向按钮切换时工具自动交换宽高
- **Claude 不要在 `gpt_prompt` 或 `prompt` 里重复写色板描述**,工具会自动加
